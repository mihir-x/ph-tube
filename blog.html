Discuss the scope of var, let and const

Variable declared with var has global scope or function scope while variables that are declared with let or const has block scope. We can update or redeclare var variables in its scope. We can not redeclare let variable but we can update it. The const variable can not be redeclared or updated. Var and let variable can be declared without initializing but const must be initialized during declaration. All of them are hoisted to the top of their scope but while var variables are initialized with undefined, let and const variables are not initialized.

Tell us the use case of null and undefined

null is a special value in javascript that represent an empty or unknown value. It indicates that a property or variable currently does not have any meaningful value.

    Use cases of null:
  • We can assign null value to variable to explicitly indicate that they have no value or should be considered empty.
  • We can use null to check if a variable or object property has any value or not
  • null is used to clear reference to an object allowing it to be garbage collected by javascript engine.
  • We can use null to reset values.


Undefined is a special value and primitive data type that indicates a variable has been declared but its value has not been assigned.
    Use cases of Undefined:
  • Undefined is often used as the starting point for variables that will be assigned values later.
  • We can use undefined as a default value for optional function parameter.
  • It is used to check if a variable is exists or if a property is defined in an object.

What do you mean by REST API

A REST API is an application programming interface that conforms to the constraints of REST architectural style and allows for interaction with RESTful web services. It is a way of accessing web services in a simple and flexible way without having to process anything. REST or Representational State Transfer is a technology that generally preferred more robust Simple Object Access Protocol(SOAP) technology because REST uses less bandwidth, simple, flexible making it more suitable for internet uses. It is used to fetch or give some information from a web services. All communication done via REST API uses only HTTP request.
